# CNOT World

## Panoramica

CNOT World è il livello infrastrutturale, geografico e sistemico dell’universo di CNOT.

Questa sezione del progetto raccoglie:
- mappe,
- infrastrutture,
- dataset reali e fittizi,
- sistemi geopolitici,
- dipendenze strategiche,
- organizzazioni,
- trasformazioni ambientali,
- reti tecnologiche,
- e strutture invisibili che modellano il mondo di CNOT.

L’obiettivo non è il realismo assoluto.

Il repository costruisce invece un ecosistema ibrido e coerente dove:
- realtà,
- speculazione,
- fiction,
- leak,
- voci,
- e immaginazione sistemica

coesistono all’interno dello stesso spazio narrativo.

# Struttura

```text
world/
│
├── gis/              # Layer geografici e dati spaziali
├── entities/         # Entità individuali (datacenter, hub, organizzazioni...)
├── systems/          # Sistemi su larga scala e dipendenze
├── infrastructure/   # Descrizioni infrastrutturali fisiche
├── timelines/        # Eventi storici e futuri
├── docs/             # Documenti e report in-universe
├── factions/         # Corporazioni, stati, gruppi e collettivi
├── overlays/         # Overlay visivi ed estetica sorveglianza
├── datasets/         # Dataset reali, fittizi e ibridi
├── renders/          # Render e output visivi
└── schema/           # Definizioni delle strutture dati
```

# Filosofia

CNOT World considera la civiltà come un sistema interconnesso.

Una città non è soltanto:
- edifici,
- strade,
- e persone.

È anche:
- reti elettriche,
- logistica alimentare,
- sistemi di sorveglianza,
- infrastrutture cloud,
- reti idriche,
- e dipendenze invisibili.

Il progetto esplora il modo in cui la vita quotidiana viene sostenuta da ecosistemi tecnici nascosti.

---

# Livelli di Canon

Le informazioni presenti possono essere:
- completamente reali,
- parzialmente fictionalizzate,
- speculative,
- trapelate,
- incerte,
- oppure volutamente ambigue.

Livelli canonici possibili:

- `official`
- `semi-official`
- `rumored`
- `leaked`
- `fictionalized`
- `unknown`

---

# Livelli di Visibilità

Entità e sistemi possono avere differenti stati di visibilità:

- `public`
- `restricted`
- `hidden`
- `classified`
- `forgotten`

---

# Formati GIS

Formati consigliati:

- GeoJSON
- GeoPackage (`.gpkg`)
- CSV
- SVG

Tutti i dati spaziali dovrebbero rimanere:
- leggibili,
- modificabili,
- versionabili,
- e compatibili con QGIS.

---

# Approccio Narrativo

CNOT World è concepito come un archivio vivente.

Il repository può contenere:
- note tecniche,
- report stile intelligence,
- mappe infrastrutturali,
- documenti interni,
- materiale di sorveglianza fittizio,
- e analisi ambientali.

Il confine tra realtà e fiction è intenzionalmente instabile.
